# balabababa 综合科研数据库自动检索系统

## 概述

本系统提供自动化的多数据库检索能力。AI根据用户需求，**通过理解用户意图**选择合适的数据库，然后加载对应的skill文件执行具体查询。

---

## 数据库索引表

| 数据库 | 目录 | 用途 | 适用场景 |
|--------|------|------|----------|
| NCBI SRA | `NCBI/NCBIdb.md` | 重测序/WGS数据检索下载 | 用户想获取原始测序数据(SRR)进行分析 |
| RiceGeneFunction | `RiceGeneFunction/ricegenefunction.md` | 水稻基因功能注释查询 | 用户想知道某个水稻基因的功能 |
| ArabidopsisGeneFunction | `ArabidopsisGeneFunction/arabidopsisgenefunction.md` | 拟南芥基因功能注释查询 | 用户想查询拟南芥基因功能或搜索特定功能相关基因 |

## AI意图判断逻辑

### 核心原则

**不是简单的关键词匹配，而是理解用户真正想做什么。**

### 意图判断流程

```
┌─────────────────────────────────────────────────────────────────┐
│                        用户请求分析                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Step 1: 理解用户想获取什么类型的数据？                            │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │ 原始测序数据？  →  用户需要SRR/SRA原始reads              │    │
│  │     ↓                                                       │    │
│  │ 是否与物种/基因组相关？                                      │    │
│  │     ↓                                                       │    │
│  │ 是下载/检索/搜索数据吗？                                     │    │
│  │     ↓                                                       │    │
│  │  → 选择: NCBI SRA                                          │    │
│  └─────────────────────────────────────────────────────────┘    │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │ 基因功能信息？  →  用户想知道某基因的功能/注释              │    │
│  │     ↓                                                       │    │
│  │ 是否是水稻基因(Os01g...格式)？                             │    │
│  │     ↓                                                       │    │
│  │  → 选择: RiceGeneFunction                                  │    │
│  └─────────────────────────────────────────────────────────┘    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 意图判断示例

#### 用户说："检索 O.meyeriana 的重测序数据"

| 分析步骤 | 判断 |
|----------|------|
| 用户想做什么？ | 获取该物种的测序数据 |
| 数据类型？ | 原始测序reads（不是基因功能） |
| 是否涉及下载/检索数据？ | 是（检索SRA数据） |
| **意图结论** | 用户需要重测序/WGS数据 → **NCBI SRA** |

#### 用户说："查询 Os03g0856500 的基因功能"

| 是否涉及基因号(Os/LOC_Os)？ | 是 |
| **意图结论** | 查询基因功能 → **RiceGeneFunction** |

## 使用方法

### AI执行流程

```
1. 读取 balabababa/Databaseinfo.md 获取数据库索引
2. 理解用户请求，判断用户意图（而非仅匹配关键词）
3. 根据意图选择目标数据库
4. 加载对应的 .md skill文件
5. 按照skill文件的流程执行查询
6. 返回结果给用户
```

### 意图判断要点

AI在判断时应考虑以下因素：

1. **数据类型**：用户想要原始测序数据，还是基因功能信息？
2. **上下文**：用户的关注点是"数据获取"还是"功能理解"？
3. **基因号格式**：是否提供了Os格式或LOC_Os格式的基因号？
4. **动词意图**：用户使用了"检索/下载/搜索"还是"查询/功能/注释"？

### 添加新数据库

如需扩展系统，按以下步骤添加：

```
1. 创建新目录: balabababa/{数据库简称}/
2. 编写 {数据库简称}db.md skill文件
3. 在 Databaseinfo.md 的索引表中添加条目
4. 在意图判断部分添加新的场景映射
```

## 数据库详情

### NCBI SRA

**路径**: `NCBI/NCBIdb.md`

**功能**: 
- 检索NCBI Sequence Read Archive (SRA)的全基因组重测序数据
- 获取SRR/DRR/ERR accession列表
- 支持按物种、测序平台、数据量等条件筛选

**适用场景**:
- 用户想获取某物种的原始测序reads
- 用户需要SRR号进行下载分析
- 用户在检索/搜索/下载SRA/WGS/重测序数据

**前置准备**: agent-browser (npm install -g agent-browser)

---

### RiceGeneFunction

**路径**: `RiceGeneFunction/ricegenefunction.md`

**功能**:
- 查询RiceData水稻基因数据库
- 获取水稻基因的功能注释信息
- 支持Os格式和LOC_Os格式基因号

**适用场景**:
- 用户想知道某水稻基因的功能
- 用户提供了Os格式基因号询问功能
- 用户在查询/获取基因注释信息

**前置准备**: agent-browser (npm install -g agent-browser)

---

### ArabidopsisGeneFunction

**路径**: `ArabidopsisGeneFunction/arabidopsisgenefunction.md`

**功能**:
- 查询TAIR和Araport拟南芥基因数据库
- 获取拟南芥基因的功能注释信息
- 支持AT格式基因号 (AT1Gxxxxx)
- 支持功能关键词搜索
- **重要**: 中文关键词需翻译为英文再查询

**适用场景**:
- 用户想知道某拟南芥基因的功能
- 用户提供了AT格式基因号询问功能
- 用户搜索特定功能相关的拟南芥基因
- 用户使用中文查询（如"耐寒性基因"、"花发育基因"）

**中文→英文翻译示例**:
| 中文 | 英文 |
|------|------|
| 耐寒性 | cold tolerance / cold stress |
| 花发育 | flower development |
| 抗逆性 | stress response |
| 干旱胁迫 | drought tolerance |
| 盐胁迫 | salt stress |

**前置准备**: agent-browser (npm install -g agent-browser)

## 扩展指南

### 扩展步骤

#### 步骤1: 创建数据库目录
```
balabababa/{数据库名称}/
```

#### 步骤2: 编写Skill文件
创建 `{数据库名称}db.md`，包含：
- **描述**: 数据库功能和用途
- **适用场景**: 何时使用此skill
- **前置检查**: 所需工具和环境
- **执行流程**: 详细的操作步骤
- **输出格式**: 结果的展示方式
- **错误处理**: 常见问题及解决方案

#### 步骤3: 更新索引
在 `Databaseinfo.md` 中添加：
1. 在"数据库索引表"中添加新条目
2. 在"意图判断示例"中添加新的场景

### Skill文件模板

```markdown
# {数据库名称} 检索技能

## 描述
{简述数据库的功能和用途}

## 适用场景
当用户意图{具体场景}时激活。

**适用场景**:
- {场景1}
- {场景2}

## 前置检查
- 检查 agent-browser 可用性
- 如未安装: npm install -g agent-browser

## 执行流程
### 步骤1: {操作}
### 步骤2: {操作}
...

## 输出格式
{结果展示格式}

## 常见问题
| 问题 | 解决方案 |
|------|----------|
| ... | ... |
```
